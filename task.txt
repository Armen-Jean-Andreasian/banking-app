Реализовать JSON API

по учету доходов и расходов пользователя

с учетом приведенных ниже требований к реализации:
    Конкурентные запросы по изменению баланса пользователя
    Выписка по всем операциям пользователя с отбором по периоду (Наименование дохода (расхода), сумма, дата),
    состояние баланса на начало и конец периода.

в качестве базы данных использовать PostgreSQL,
кешировать информацию о пользователе на 20 минут
    по критериям поиска(то есть если запросили начало и конец периода для конкретного пользователя,
    то в кеше это должено храниться 20 минут и за эти 20 минут все запросы на этого пользователя
    с данными критериями должны получать ответ из кеша)

нужно сделать все максимально оптимально, что-бы Time complexity не превышало O(n log(n))
